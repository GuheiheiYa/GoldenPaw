## 二、功能缺失清单（按优先级）

### 🔴 P0 — 核心功能缺失

1. **交易编辑功能** ✅ 已修复
   - `transactionStore` 已添加 `updateTransaction()`
   - 明细页点击编辑打开 RecordSheet 并回显数据
   - `appStore` 支持 `editingTransactionId` 状态管理

2. **交易删除功能（明细页交互）** ✅ 已修复
   - 明细页长按触发 `uni.showModal` 确认删除
   - 调用 `transactionStore.deleteTransaction()`

3. **收支报表真实数据** ✅ 已修复
   - `report.vue` 接入 `transactionStore` 和 `categoryStore`
   - 周期切换（本周/本月/本季/本年）动态过滤
   - 柱状图、饼图、排行全部动态计算

4. **AI记账核心功能** ✅ 已修复
   - 输入框绑定 `v-model`，支持发送和回车
   - 本地规则解析器：识别金额、分类、收支类型
   - 支持自然语言记账（"早餐15块"）和智能分析（"本月花了多少"）
   - 聊天消息实时更新，快捷操作可触发

5. **存钱目标真实数据** ✅ 已修复
   - 新建 `goalBudgetStore`（Pinia + persist）
   - 目标支持新建、存入金额、调整目标
   - 预算按分类设置，used 实时联动 `transactionStore`
   - 弹窗表单支持完整 CRUD

### 🟡 P1 — 重要体验缺失

6. **首页预算 & 存钱目标接入 Store** ✅ 已修复
   - `index.vue` 预算和目标从 `goalBudgetStore` 动态读取
   - 预算 used 实时计算，目标进度动态展示

7. **资产趋势图表真实数据** ✅ 已修复
   - `assets.vue` 趋势图从硬编码改为基于 `transactionStore` 动态计算最近6个月净资产
   - 月度净资产变化也改为动态计算

8. **资产环形图渲染问题** ⚠️ 已知限制（未修复）
   - 使用多个 `conic-gradient` 绝对定位叠加，实际会互相覆盖
   - 需要改为 SVG 环形图（参考 goals.vue 的实现方式）

9. **待确认 → 真实交易** ✅ 已修复
   - `pending.vue` 点击"确认"解析数据并调用 `transactionStore.addTransaction()` 创建真实交易
   - 同时更新对应账户余额
   - "编辑"按钮已实现弹窗编辑（金额/商户/分类）
   - "忽略"按钮添加确认弹窗

10. **明细页 FAB 添加按钮** ⚠️ 未实现（非核心）
    - 设计稿右下角有 + 按钮，可通过 TabBar 中间记账按钮或长按删除旁的编辑替代

11. **明细页 highlight 参数未使用** ✅ 已修复
    - `detail.vue` 读取 `highlight` 参数并添加脉冲高亮样式
    - 3 秒后自动清除高亮

### 🟢 P2 — 体验优化 & 细节修复
