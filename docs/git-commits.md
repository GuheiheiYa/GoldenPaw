## 提交规范（后续遵循）

```
feat: 简短描述
- 需求: R-xxx, R-xxx
- 问题: ISS-xxx（如解决）
- 文件: 修改的主要文件
```

**类型前缀**：
- `feat:` 新功能
- `fix:` Bug 修复
- `refactor:` 重构（无功能变更）
- `docs:` 文档更新
- `style:` 纯样式调整
- `test:` 测试相关
